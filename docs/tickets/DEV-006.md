# DEV-006: Fix Cloud Run Pony Worker Infrastructure Deployment

**Phase**: 5 (Execution Blocker)
**Assignee**: DevOps Engineer / Lead Dev
**Status**: To Do

## Bug / Blocker Description
An attempt was made to run the Phase 5 end-to-end scene generation test against the real **Stable Diffusion Pony Worker** Cloud Run endpoint (`pony-xl-inference-service`). The test failed immediately because the Cloud Run infrastructure does not exist in the active `vertex-ai-489604` GCP project. 

The `run.googleapis.com` API is disabled, indicating that the `main` branch Terraform CI/CD pipeline did not execute successfully.

## Root Cause Analysis
Detailed error analysis on the failing components reveals three critical blockers in the `main` branch preventing the deployment of the Cloud Run worker:

1.  **Broken CI/CD WIF Authentication**: The `.github/workflows/deploy-infrastructure.yml` file contains a dummy placeholder for the Google Cloud Workload Identity Provider (`projects/123456789/locations/global...`).
2.  **Incorrect Project Context**: The CI/CD workflow is hardcoded to use a Service Account from an older, abandoned project: `github-actions-cd-sa@vertex-pipeline-awfg2.iam.gserviceaccount.com`. It needs to be updated to target `vertex-ai-489604`.
3.  **Missing Artifact Registry Image**: The `cloudrun.tf` Terraform code relies on the container image `us-central1-docker.pkg.dev/${var.project_id}/repo/pony-xl-inference:latest`. This image has never been built or pushed to the registry. A theoretical `terraform apply` would fail because Cloud Run validates the existence of the container during provisioning.

## Acceptance Criteria for Fix
*   Update `.github/workflows/deploy-infrastructure.yml` with the correct `workload_identity_provider` ID for `vertex-ai-489604`.
*   Update the WIF Service Account email to point to the active project.
*   Implement a Docker build step/script to build and push the `pony-xl-inference:latest` container to Artifact Registry *before* triggering the Terraform apply.
*   Successfully execute the CI/CD pipeline so `pony-xl-inference-service` is live in Cloud Run.

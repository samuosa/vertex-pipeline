# QA-001: Draft comprehensive test plan

**Phase**: 5
**Assignee**: QA Engineer
**Status**: In Progress

## 1. Functional API Criteria
- **Endpoint**: Cloud Run HTTPS endpoint for image generation.
- **Payload Validation**: Ensure POST requests containing standard Diffusers/vLLM generation parameters reject invalid parameters (400 Bad Request).
- **Response Validation**: Ensure 200 OK responses return proper output image URLs (mapped from the output bucket) within the payload, and errors (like out of memory) are handled gracefully (500 or 503).

## 2. Performance Criteria
- **Cold-Start Latency**: Given the use of GCSFuse and Image Streaming, cold starts for the L4 GPU container must complete within 15-30 seconds.
- **Sustained Throughput**: Since `concurrency=1` is strictly enforced to prevent OOM errors, throughput scales horizontally. We expect successful concurrent requests scaling up to the quota limit without failing.
- **Load Testing**: A k6 script `tests/k6/load_test.js` is developed to simulate concurrent users and measure cold-start vs warm-start latencies.

## 3. Cost Validation
- **Scale-to-Zero Verification**: After load tests conclude, we will monitor Stackdriver/Cloud Monitoring metrics to ensure the instance count drops to 0 after the idle timeout period.
- **Budget Caps**: Ensuring that any looping or stalled requests timeout to avoid unlimited L4 GPU billing.
